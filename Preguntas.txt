1¿Qué es Next.js y por qué se dice que es el framework de React más popular?
Es un framework de React que permite construir aplicaciones con renderizado en el servidor, optimización, fetching de datos y rutas de API. Es la más popular porque facilita el desarrollo de aplicaciones como Twitch, Netflix, TikTok, etc.

2 ¿Cuál es el comando exacto con npx para inicializar un proyecto de Next.js asegurando usar la última versión?
npx create-next-app@latest

3 ¿Cuál es el comando utilizado para levantar el entorno de desarrollo y en qué puerto se ejecuta por defecto? 
npm run dev y se ejecuta en el puerto 3000

4 ¿Qué función cumple el comando npm run build en un proyecto de Next.js? 
Se utiliza para crear y empaquetar la aplicación para su despliegue en producción

5 Describe para que sirven las carpetas, public y app
Public: Amacena archivos estáticos como imágenes, vídeos y audios que se sirven directamente en la web.
APP: Es la carpeta más importante donde se definen las rutas, la lógica de la aplicación y los componentes, utilizando el sistema de enrutado basado en archivos.

6 ¿Qué es un CCS MODULE y como se usan? 
Es una forma nativa de cargar estilos usando la extensión .module.css. Se usan importando los estilos como un objeto y aplicándolos mediante Styles.nombreClase, lo que genera un identificador único (hash) para evitar colisiones entre clases.

7 ¿Qué hace el componente  Layout.tsx ? 
Sirve para envolver las páginas y rutas. Permite compartir elementos de la interfaz entre múltiples páginas sin que estos se vuelvan a renderizar al navegar, manteniendo su estado.

8 ¿Qué es el enrutamiento ?
El sistema permite definir y navegar a través de diferentes segmentos o rutas de una página web.

9 ¿Cuál es la diferencia entre Image e img?
Image ofrece optimizaciones automáticas como ajuste de tamaño, cambiar el formato al más óptimo, evita saltos de renderizado y utiliza Lazy Load por defecto.
Img puede causar problemas de rendimiento y mayor consumo de ancho de banda.

10 ¿Qué es el Lazy Load?
Es una técnica de optimización donde los recursos no se cargan hasta que son visibles en el viewport del usuario, ahorrando ancho de banda inicial.

11 Cual es el funcionamiento del siguiente código, en que momento se muestra cada imagen.
Es un código para alterar la visibilidad de imágenes según el tamaño de la pantalla.
Imagen Desktop (hero-desktop.png) tiene la clase hidden md:block. Esto significa que permanece oculto en dispositivos móviles y solo se muestra para pantallas de 768px aproximadamente en adelante.
Imagen Mobile (hero-mobile.png) tiene la clase block md:hidden. Esto hace que la imagen sea visible para dispositivos móviles y se oculta cuando sobrepasa el tamaño md.

12 ¿Cómo funciona el enrutamiento en Next JS  ? 
Funciona a través del File System Routing. Cada carpeta dentro de la carpeta app representa un segmento de la URL, y el archivo page.tsx dentro de esa carpeta es el que hace pública esa ruta.

13 ¿Qué es una ruta en el contexto web y donde las hemos visto en la clase ? 
Es una dirección dentro de un sitio. En clase lo hemos visto en el git el repositorio es una ruta en el sistema de archivos.

14 ¿Qué arrojo a MiduDev el localhost:300/dashboard, donde hemos visto esos errores en la clase ? 
Arrojo un error 404 porque la ruta aún no había sido creada mediante una carpeta de archivos. En clase lo hemos visto cuando hacíamos peticiones en la API y poníamos por ejemplo el URL mal.

16 ¿Qué  es el File System Routing?
Es un sistema en donde el enrutado está basado en el sistema de archivos; la estructura de carpetas y archivos en el disco define automáticamente las rutas disponibles en la aplicación.

17 ¿Cómo funcionan las rutas anidadas?
Se crean colocando carpetas dentro de otras carpetas. La URL resultante replica esa estructura jerárquica.

18 ¿Investiga por que es necesario esto ?
Los layouts y las rutas anidadas son fundamentakes para la eficiencia de la aplicación porque permite reutilizar partes de la interfaz que no deben cambiar mientras el usuario navega entre diferentes secciones.

19 ¿Para que sirve el componente Link?
Sirve para realizar navegaciones por parte del cliente. Permite cambiar de página sin recargar todo el sitio, haciendo que la transición sea instantánea y fluida

20 ¿Qué se carga cuando usamos <a> y que cuando se usa <link> ?
<a>.Se carga toda la página y todos sus recursos desde 0
<link>.Solo se carga el contenido mínimo necesario de la nueva ruta, reutilizando el código y estado que ya están cargados.

21 ¿Investiga que es un Hook?
Son funciones que permiten acceder a información específica del framework, como datos de las rutas, el historial o el estado del router.

22 ¿Qué hooks usa Midu?
usePathname para leer la ruta actual en la que se encuentra el usuario.

23 ¿Por que ocurre el siguiente error?
Porque el hook usePathname solo puede ejecutarse en Client Components.

24 ¿Es posible ejecutar los componentes de react en el cliente y el servidor?
Si. Los componentes se oueden renderizar en el servidor y luego cobrar vida en el cliente para manejar interactividad y eventos.

25 ¿Qué es el code splitting?
Es el proceso de dividir el código de la aplicación en partes pequeñas.
